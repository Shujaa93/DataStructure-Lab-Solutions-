                                  #  Project: The Time-Stream Stabilizer
Welcome, Time Agent.
You work for the Chronos Corporation. The fabric of the universe is collapsing because historical events 
are becoming disordered. To save existence, we have digitized the timeline into a Binary Search Tree (BST).
Your mission is to build the Time-Stream Stabilizer software. You must inject events into their correct
chronological positions, remove paradoxes, and report the status of history.

# The Data Structure
The timeline is strictly linear. You must implement a BST where every node is a Time Capsule:

struct TimeCapsule {
    int year;               // The Key
    string eventName;       // The Data
    TimeCapsule* left;      // Earlier years
    TimeCapsule* right;     // Later years
};

The Laws of Time (BST Properties) :
Chronological Order: All years smaller than the current node must be in the Left Subtree.
Future Progression: All years larger than the current node must be in the Right Subtree.
Uniqueness: No two events can happen in the exact same year (Duplicate keys are not allowed).

Required Commands :
Your system must accept the following command inputs:
Command	BST Operation	Description
INJECT <Year> <Name>	Insert	Insert a new historical event. If the year exists, do nothing.
PARADOX <Year>	Delete	A time paradox has occurred! Remove this year from the tree. (Handle cases with 0, 1, or 2 children).
SEARCH <Year>	Search	Check if an event exists in a specific year. Print the event name if found.
REPORT	In-Order Traversal	Print the entire timeline in chronological order (Ascending).
